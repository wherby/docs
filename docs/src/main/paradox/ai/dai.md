昨天睡觉前我突然想起用算法的思路来描述我的看法。

借用数据容量单位，每一级1024倍

B、KB、MB、GB、TB、PB、EB、ZB、YB

大模型解决的问题我们看做一个高阶的模式匹配或者搜索查询。
对于世界上的知识我们认为是无限的，会以YB级别衡量的点。
现有的大模型是基于EB级别的知识点，形成了TB级别的特征点。

我认为将来解决问题的关键不在于大模型，而在于大模型辅助形成的专业模型

## 大模型的限制

```
现在AI是做什么事情，简单的来说是还是在做search的事情，所谓的大模型，也是在一个可以认为是无限的高保真度的照片里查询模式匹配的片段.
虽然LLM能得到所谓的推理能力可以handle未见过的情况判定，但是LLM所形成的模型所形成的推理能力不能得到外部输入影响进行进化
```

就如同上面提到的，大模型训练的样本最大的集合是现实世界的snapshot,但是所谓知识是会随时间增加的，况且还有很多的行业知识在深层的网络中或者未经数字化，这些新的知识很难反馈影响到已经训练好的模型。

换另一个视角，所谓的行业知识其实比大模型训练的样本小得多，比如金融里会计或者税务的知识set可能就是GB到TB级别的，我们如果要获得具体知识的查询，我们没有必要维持一个TB级别的大模型。

## 大模型辅助的智能体进化

怎么来设计一个可以自己进化的行业知识智能体呢？
这里有个假设，在任何一个行业知识（Domain knowledge）相对小的，这也是显而易见的。
所以我们可以维护一个行业知识的集合，利用当前集合的知识来训练进化当前的行业知识模型。
而大模型的作用仅在该智能体不能解决的时候使用，并生成新的知识点加入该智能体。

这样就形成了大模型作为辅助的行业知识智能体进化的整个循环。

该智能体对于查询，只需要维护相对小的模型体积，而大模型的依赖只会在新的知识点被发现的时候使用。

```
就像在web2.0时代的Stack Overflow 和Google， Google就像大模型具有全部网页的知识，然而专业知识在Stack Overflow 会更准确和及时获得。
```

## 大模型自身的进化

大模型可以提取出一个Common Sense的小模块，此模块用来激活各个Expert模块，并且记录用户的反馈作为新的知识训练集，然后迭代训练各个Expert模块。
